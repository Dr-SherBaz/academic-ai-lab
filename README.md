# Academic AI Lab

Static Cloudflare Pages site with Cloudflare Pages Functions scaffolding for a WhatsApp-first managed academic, research, AI, and digital-workflow service platform.

## Local Development

This is a plain HTML/CSS/JS project. You can open `index.html` directly for frontend review.

For Cloudflare Pages Functions, use Wrangler later:

```bash
npx wrangler pages dev .
```

## Git and Cloudflare Deployment

The repository is Git-connected. Cloudflare Pages should deploy from GitHub according to the existing Cloudflare project settings. This repo does not change DNS, Cloudflare project settings, email routing, or external deployment settings.

Do not commit secrets. Do not push unless explicitly requested.

## Decap CMS — Public Content Management

Decap CMS is configured for editing **public website content only** (blog posts, resources, templates, case notes, workshops, educational resources).

**WARNING:** Decap CMS must NOT be used for:
- Client order data, uploaded files, or payment information
- Magic-link tokens, private dashboard data, or expert assignment records
- WhatsApp project updates, .env files, or any secrets
- Those are managed through Cloudflare Pages Functions, D1, R2, and the secure dashboard.

### Content Folder Structure

```
content/
  blog/                  → Markdown source for Insights & Guides
  templates/             → Markdown source for Templates & Downloads
  workshops/             → Markdown source for Workshop Announcements
  case-notes/            → Markdown source for Case Notes / Work Samples
  resources/             → (Reserved)
  educational-resources/ → (Reserved)
```

### Media Handling

- `media_folder: "assets/uploads"` — for public blog/resource images only
- `public_folder: "/assets/uploads"` — served at `/assets/uploads/`
- Do NOT upload client files, private documents, or sensitive materials here

## Adding Content Without Editing Code

### Option A — Git Folder Method (works now, no auth required)

1. Copy an existing Markdown file from `content/blog/`, `content/templates/`, etc.
2. Edit the frontmatter (title, date, slug, excerpt, category, tags) and body
3. Set `status: published` when ready
4. Run the build script:
   ```bash
   node scripts/build-content.js
   ```
5. Commit and push to GitHub — Cloudflare Pages redeploys automatically

### Option B — Decap CMS Method (requires GitHub OAuth setup)

1. Open `/admin` on your live site
2. Log in through GitHub authentication (must be configured first — see below)
3. Create or edit posts through the visual editor
4. Click **Publish** — changes are committed to GitHub
5. Cloudflare Pages redeploys automatically

### GitHub Authentication Setup

The `admin/config.yml` uses `backend: github` with the repository and branch
configured. Before live browser editing works, you need to set up one of these
authentication paths:

**Path A — Decap OAuth with GitHub (recommended)**
Decap CMS requires an OAuth provider for GitHub authentication. Use the
official `netlify-cms-github-oauth-provider` (works outside Netlify) or
deploy a lightweight OAuth handler. See:
https://decapcms.org/docs/github-backend/

**Path B — Cloudflare Pages Function OAuth helper (future)**
A lightweight Cloudflare Pages Function can act as the OAuth callback
handler, removing the need for a separate OAuth server. This can be
added later in `functions/api/oauth/`.

**Path C — Git-folder method only (works immediately)**
You do not need browser-based login to use Decap CMS. Use the Git-folder
method (Option A above) — create Markdown files directly, run the build
script, commit, and push. This works today with no additional setup.

### Security Notes for `/admin`

- `/admin` is only for public content editing
- Do not store secrets in `admin/config.yml`
- Do not allow Decap to edit backend functions, `.env` files, order records,
  or private client data
- The `assets/uploads/` media folder is for public images only — not client files
- GitHub authentication/OAuth must be configured before any browser-based
  CMS editing goes live

## Build Scripts

```bash
# Generate HTML pages and index JSON from Markdown content
npm run build:content

# (alias)
npm run dev:content
```

The build script:
- Reads Markdown files from `content/` folders
- Parses YAML frontmatter
- Skips drafts where `status` is not `published`
- Generates individual HTML pages into `blog/`, `templates/`, `case-notes/`, `workshops/`
- Writes `data/blog-index.json` (master index) and per-collection index files
- Sorts newest first
- Preserves service CTA buttons on every content page

Content pages are generated with full site headers, navigation, footers,
and styling that matches the main site design.

If no build pipeline is configured on Cloudflare Pages, run the command
manually before committing.
